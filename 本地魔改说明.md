# 本地魔改说明

## 1. 定位

这个目录保存的是 `Uptime Kuma Remastered` 的完整本地源码。

它和官方上游的关系是：

- 基础版本来自 `Uptime Kuma 2.2.1`
- 保留原本监控、状态页、通知、Cloudflared 等基础能力
- 在此基础上加入适合当前个人部署环境的定向魔改

这不是一个“通用发行版”，而是一个：

- 个人长期维护分支
- 面向当前实际线路节点与玩家展示需求的版本
- 允许为体验做较强主观设计取舍的独立实现

## 2. 主要魔改方向

### 2.1 后台运维体验

- 监控状态变化改为消息箱，不再用右下角连发 toast
- 避免大规模波动时挡住后台按钮

### 2.2 状态页排序控制

- 默认可按状态降序显示
- 可关闭默认排序，恢复手动固定顺序

### 2.3 状态页展示层

- 可选覆盖原版状态页渲染
- 支持推荐卡片
- 支持分区卡片
- 支持列数控制
- 支持节点标签与备注
- 支持主展示节点选择

### 2.4 编辑器增强

- 侧栏更宽
- 可拖拽改宽度
- 卡片配色可选
- 分区小标题可单独编辑

### 2.5 快照持久化

- 保存状态页时自动生成本地快照
- 避免重建或误改后无从恢复

## 3. 与正式环境的关系

正式运行目录：

- `D:\_Plana_Docker\UptimeKuma`

正式数据目录：

- `D:\_Plana_Docker\UptimeKuma\data`

本目录只负责：

- 维护源码
- 构建镜像
- 提供恢复与导出工具

## 4. 注意事项

- 不要直接修改容器内文件
- 不要在容器运行时直接写 SQLite 数据库
- 需要改状态页配置时，优先走页面保存
- 如果必须离线改库，先停容器
- 如果以后继续做 UI 改造，优先保留“可配置化”思路，不再回退到硬编码
